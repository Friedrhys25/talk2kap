# Firebase Database Structure Guide - Emergency Hotlines

## Overview
This document explains the Firebase Firestore database structure used in the `emergency.jsx` component. Use this guide to understand how to fetch, create, update, and delete emergency hotline data from other systems.

---

## 📊 Database Collection Structure

### Collection: `emergencyHotlines`
Stores all emergency hotline information (Police, Fire, Medical) that are displayed to app users.

**Path:** `firestore.collection("emergencyHotlines")`

---

## 📋 Document Schema

Each document in the `emergencyHotlines` collection has the following structure:

```
emergencyHotlines/
└── [Document ID] (auto-generated by Firebase)
    ├── name: String
    ├── number: String
    ├── category: String ("Police" | "Fire" | "Medical")
    ├── createdAt: Timestamp
    └── updatedAt: Timestamp (only on updates)
```

### Field Definitions

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `id` | String | Auto-generated Document ID | `"doc123abc456"` |
| `name` | String | Agency or hotline name | `"San Roque Police Station"` |
| `number` | String | Contact phone number | `"0917-123-4567"` or `"911"` |
| `category` | String | Type of service (Police, Fire, or Medical) | `"Police"` |
| `createdAt` | Timestamp | When the hotline was created | `2026-04-08T10:30:00Z` |
| `updatedAt` | Timestamp | When the hotline was last modified (on update only) | `2026-04-08T10:30:00Z` |

### Example Document
```json
{
  "id": "emg_001_police",
  "name": "San Roque Police Station",
  "number": "0917-123-4567",
  "category": "Police",
  "createdAt": "2026-04-08T10:30:00Z",
  "updatedAt": "2026-04-08T11:45:00Z"
}
```

## 🔍 How to Fetch Data

### Method 1: Real-time Listener (Recommended for UI)
Listens to all hotlines in real-time and updates whenever data changes:

```javascript
import { getFirestore, collection, onSnapshot } from "firebase/firestore";
import app from "./firebaseConfig";

const firestore = getFirestore(app);

const ref = collection(firestore, "emergencyHotlines");

const unsub = onSnapshot(
  ref,
  (snap) => {
    const hotlines = snap.docs.map((d) => ({
      id: d.id, // Document ID
      ...d.data() // All fields (name, number, category, createdAt, updatedAt)
    }));
    
    console.log("Hotlines:", hotlines);
    // Use hotlines array here
  },
  (error) => {
    console.error("Error fetching hotlines:", error);
  }
);

// Clean up listener when done
// unsub();
```

### Method 2: One-Time Read
Fetch hotlines once (without listening for changes):

```javascript
import { getFirestore, collection, getDocs } from "firebase/firestore";

const firestore = getFirestore(app);

const querySnapshot = await getDocs(collection(firestore, "emergencyHotlines"));
const hotlines = querySnapshot.docs.map((d) => ({
  id: d.id,
  ...d.data()
}));

console.log("Hotlines:", hotlines);
```

## ➕ How to Create Data

Add a new emergency hotline:

```javascript
import { getFirestore, collection, addDoc, serverTimestamp } from "firebase/firestore";

const firestore = getFirestore(app);

const newHotline = {
  name: "San Roque Fire Department",
  number: "0917-987-6543",
  category: "Fire",
  createdAt: serverTimestamp() // Automatically set to server time
};

const docRef = await addDoc(collection(firestore, "emergencyHotlines"), newHotline);
console.log("Hotline created with ID:", docRef.id);
```

**Required Fields for Creation:**
- `name` (String, non-empty)
- `number` (String, non-empty)
- `category` (String: "Police", "Fire", or "Medical")

**Auto-Generated Fields:**
- `createdAt` (Timestamp, set via `serverTimestamp()`)

## ✏️ How to Update Data

Update an existing hotline:

```javascript
import { getFirestore, doc, updateDoc, serverTimestamp } from "firebase/firestore";

const firestore = getFirestore(app);
const docId = "emg_001_police"; // Document ID

const updatedFields = {
  name: "San Roque Police Station Updated",
  number: "0917-999-8888",
  category: "Police",
  updatedAt: serverTimestamp() // Always update this timestamp
};

await updateDoc(doc(firestore, "emergencyHotlines", docId), updatedFields);
console.log("Hotline updated successfully");
```

**Notes:**
- Always include `updatedAt: serverTimestamp()` when updating
- Only include fields you want to change (partial update is fine)
- Document ID is required

---

## 🗑️ How to Delete Data

Delete a hotline:

```javascript
import { getFirestore, doc, deleteDoc } from "firebase/firestore";

const firestore = getFirestore(app);
const docId = "emg_001_police";

await deleteDoc(doc(firestore, "emergencyHotlines", docId));
console.log("Hotline deleted successfully");
```

## 🔐 Firestore Rules (Security)

Ensure your Firestore security rules allow appropriate access. Example rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow all users to read emergency hotlines (public data)
    match /emergencyHotlines/{document=**} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.token.admin == true;
    }
  }
}
```

4. **CRUD Operations**:

## ⏱️ Timestamp Fields

### `createdAt`
- **Type**: Firestore Timestamp
- **Set On**: Document creation only (via `serverTimestamp()`)
- **Immutable**: Yes, should not be changed
- **Use Case**: Audit trail, sorting by creation date

### `updatedAt`
- **Type**: Firestore Timestamp
- **Set On**: Every document update (via `serverTimestamp()`)
- **Immutable**: No
- **Use Case**: Track when hotline info was last modified

## 📝 Complete Example: Calling from Another System

```javascript
import { initializeApp } from "firebase/app";
import { getFirestore, collection, onSnapshot, addDoc, updateDoc, deleteDoc, doc, serverTimestamp } from "firebase/firestore";

// Initialize Firebase (use your config)
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_AUTH_DOMAIN",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_STORAGE_BUCKET",
  messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
  appId: "YOUR_APP_ID"
};

const app = initializeApp(firebaseConfig);
const firestore = getFirestore(app);

// 1. READ ALL HOTLINES (Real-time)
function subscribeToHotlines(callback) {
  return onSnapshot(collection(firestore, "emergencyHotlines"), (snap) => {
    const hotlines = snap.docs.map((d) => ({ id: d.id, ...d.data() }));
    callback(hotlines);
  });
}

// Usage:
const unsubscribe = subscribeToHotlines((hotlines) => {
  console.log("Current hotlines:", hotlines);
  // hotlines is an array of hotline objects
});

// 2. CREATE NEW HOTLINE
async function addNewHotline(name, number, category) {
  try {
    const docRef = await addDoc(collection(firestore, "emergencyHotlines"), {
      name,
      number,
      category,
      createdAt: serverTimestamp()
    });
    console.log("Created with ID:", docRef.id);
    return docRef.id;
  } catch (error) {
    console.error("Error creating hotline:", error);
  }
}

// Usage:
// addNewHotline("San Roque Police", "0917-111-1111", "Police");

// 3. UPDATE HOTLINE
async function updateHotline(docId, updates) {
  try {
    await updateDoc(doc(firestore, "emergencyHotlines", docId), {
      ...updates,
      updatedAt: serverTimestamp()
    });
    console.log("Updated successfully");
  } catch (error) {
    console.error("Error updating hotline:", error);
  }
}

// Usage:
// updateHotline("doc123", { number: "0917-222-2222" });

// 4. DELETE HOTLINE
async function deleteHotline(docId) {
  try {
    await deleteDoc(doc(firestore, "emergencyHotlines", docId));
    console.log("Deleted successfully");
  } catch (error) {
    console.error("Error deleting hotline:", error);
  }
}

// Usage:
// deleteHotline("doc123");

// Cleanup when needed
// unsubscribe();
```

## ❓ FAQ

**Q: Can I query hotlines by category?**
A: Yes, use Firestore queries:
```javascript
const q = query(collection(firestore, "emergencyHotlines"), where("category", "==", "Police"));
const snapshot = await getDocs(q);
```

**Q: Can I sort hotlines?**
A: Yes, use ordering:
```javascript
const q = query(collection(firestore, "emergencyHotlines"), orderBy("name", "asc"));
```

**Q: How do I handle timestamps?**
A: The `createdAt` and `updatedAt` fields are Firestore Timestamps. Convert to JavaScript Date:
```javascript
const createdDate = hotline.createdAt.toDate();
```

**Q: Can I batch update multiple hotlines?**
A: Yes, use `writeBatch()`:
```javascript
const batch = writeBatch(firestore);
// Add multiple operations
await batch.commit();
```
